Essential 8 vs. The BoM Attack

Mapping Controls to a Real-World Breach

How would the Essential 8 have fared against the Bureau of Meteorology attack?

Click on each control to analyse its potential impact.

Application Control

Prevent execution of unapproved/malicious programs, scripts, installers, etc.

Observed Failure:

A password dumping utility and a Remote Access Tool (RAT) were executed on the network. These are unauthorised, malicious applications.

Protective Impact:

Proper application control would have blocked these tools from running, preventing the attacker from stealing credentials and establishing persistent access. This single control could have stopped the breach from escalating.

Patch Applications

Remediate security vulnerabilities in applications within 48 hours for extreme risks.

Observed Failure:

The report notes adversaries use exploits for known vulnerabilities. While the exact entry point wasn't specified, the presence of multiple malware types (state-sponsored RAT + cybercrime ransomware) suggests a target-rich environment, likely with unpatched applications.

Protective Impact:

Timely patching reduces the attack surface. If the initial entry was via an application vulnerability (e.g., in a web server or PDF reader), this control would have prevented it entirely.

Configure Microsoft Office Macros

Block macros from the internet and only allow vetted macros to run.

Observed Failure:

The network was infected with CryptoLocker ransomware. At the time, a primary delivery mechanism for this malware was malicious Office macros in phishing emails.

Protective Impact:

Blocking untrusted macros would have neutralised the ransomware threat, which the report identified as the "most significant threat to the Bureau's data retention and continuity of operations."

User Application Hardening

Configure web browsers to block Flash, ads, and Java. Disable unneeded features in other apps.

Observed Failure:

The report doesn't confirm this vector for the BoM, but it explicitly states other government agencies were being compromised via Adobe Flash exploits on "watering hole" websites during the same period.

Protective Impact:

This is a key preventative measure. By hardening browsers, an entire class of "drive-by download" attacks is eliminated, reducing the chance of an initial compromise without any user interaction.

Restrict Administrative Privileges

Use the principle of least privilege. Don't use admin accounts for general tasks.

Observed Failure:

This was a critical failure. The attacker gained control of and used a legitimate domain administrator account, giving them the "keys to the kingdom" to access domain controllers and file servers.

Protective Impact:

Restricting admin rights makes it much harder for an attacker to escalate privileges and move laterally. Even if they compromise a standard user account, they can't easily access critical systems or deploy network-wide malware.

Patch Operating Systems

Remediate OS security vulnerabilities in 48 hours for extreme risks.

Observed Failure:

The report concluded security controls were "insufficient". This strongly implies that basic hygiene, like OS patching, was likely lacking. An unpatched OS could have allowed the initial malware to run or enabled privilege escalation.

Protective Impact:

A fully patched OS is much more resilient. This closes known security holes that malware and attackers rely on to gain a foothold and elevate their access within a system.

Multi-factor Authentication (MFA)

Require a second proof of identity for users, especially for remote access and privileged accounts.

Observed Failure:

The attacker used a password dumping utility to steal credentials and then used a legitimate administrator account. This suggests access was protected only by a password.

Protective Impact:

MFA would have rendered the stolen password useless. Even with the correct credentials, the attacker would not have been able to log in to the admin account without the second factor (e.g., a code from a phone app), stopping the attack cold.

Regular Backups

Create, maintain, and test backups of important data that are stored offline or disconnected.

Observed Failure:

The CryptoLocker ransomware on the network posed a major threat to data availability. The primary defense against ransomware is restoring from a clean backup.

Protective Impact:

With regular, tested, and offline backups, the impact of the ransomware would have been minimal. Instead of considering paying a ransom or losing data permanently, the Bureau could have wiped the affected systems and restored the data, turning a major crisis into a manageable inconvenience.
